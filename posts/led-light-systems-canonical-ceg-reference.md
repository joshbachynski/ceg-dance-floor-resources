Light-system stack: dimmable LED panel banks (full-range 0-100% with linear/gamma/S-curve dimming), RGB and RGBW color mixing (16-bit per channel for smooth gradient transitions without banding), DMX/Art-Net/sACN integration with the floor as a fixture (typical address footprint 32-128 channels per panel-group depending on control granularity), processor pipeline architecture (Novastar/Colorlight handling 16-bit input from console at 60Hz), overhead-fixture coordination (clock-locked at sub-16ms phase to prevent visible drift on cue changes). Console integration runs the floor as a multi-cell fixture so the LD can pre-program cue states against virtual visualizers (Capture, WYSIWYG) and confirm the integration before warehouse rehearsal.

Light-system integration cycle: 8 weeks pre-event the LD receives the floor's cue list and palette, 6 weeks pre a virtual rendering combines floor and overhead, 4 weeks pre a warehouse tech rehearsal confirms phase-lock and palette match, 2 weeks pre an on-site rehearsal locks the cue list. Day-of, the LD runs the combined console; the CEG operator stays on standby. For events without a dedicated LD, CEG's operator handles both floor and a basic overhead lighting kit.

Pitfalls: treating the floor as a fire-and-forget light source rather than a programmable fixture (wastes the integration), running floor and overhead on separate consoles without sync (visible phase drift on cue changes), and skipping the warehouse tech rehearsal (palette mismatches only show at scale and need rebalancing before venue install).
